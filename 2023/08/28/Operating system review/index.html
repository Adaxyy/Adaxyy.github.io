3 Process 进程

3.1 Process 进程

3.1.1 进程的定义

进程是现代分时操作系统的工作单元。[*部分情况下job和process可以互换]

  • 进程包括:

    • 程序代码/文本段text section/代码段 code section

    • 程序计数器 program counter

    • 堆栈 stack 临时数据(函数参数、返回地址、局部变量)

    • 数据段 data section: 全局变量

    • 可包含堆 heap: 动态分配的内存

3.1.2 进程状态

  • 进程状态 process status

    • new

    • running

    • blocked/Waiting: 等待发生某个事件/信号

    • ready: 等待分配处理器

    • terminated: 完成执行

      在一个时间点, 只有一个进程可以在一个处理器上running, 但是可以有很多进程处于ready或waiting状态

3.1.3 进程控制块 PCB

PCB process control block包含许多与某个特定进程相关的信息, 每次进行中断或程序调用时, 将会保存到PCB或从PCB重新加载状态

  • 进程状态Process state: 如3.2所述
  • 程序计数器 Program counter(PC)
  • CPU寄存器 CPU register: 各种寄存器的具体信息和数值, 在发生中断时需要和PC一样保存
  • CPU调度信息(CPU-scheduling information): 包括进程优先级、调度队列指针和其他参数
  • 内存管理信息(Memory management info)
  • 记账信息(Accounting info)
  • I/O状态信息

3.2 进程调度

多道程序设计目标: 无论何时都有进程运行, 从而最大化CPU利用率

分时系统的目的: 在进程间快速切换CPU

进程调度器选择一个可用进程到CPU上运行

3.2.1 Scheduling queue 调度队列

进程加入系统时, 被加入作业队列job queue, 这个队列包括系统

ready queue

device queue

3.2.2 调度程序

使用不同的调度器(scheduler)

  • long-term scheduler/job scheduler从磁盘的缓冲池选择进程, 加载到内存 [频率低, 效率低]
  • short-term scheduler/CPU schedule从准备执行的进程中选择进程, 分配cpu [频率高, 效率高]

进程分类

  • I/O密集型进程 I/O-bound process
  • CPU密集型进程 CPU-bound process

部分系统可能直接没有long-term S, 全部都放在CPU; 而有的会引入额外的中期调度程序 medium-term scheduler, 可以将进程从CPU中移出, 并可能从中断处继续执行(交换swap out or in)

3.2.3 Context switch 上下文切换

使用PCB表示进程上下文, 执行状态保存 state save和状态恢复 state restore

上下文切换时纯粹的时间消耗, 具体时间与硬件支持密切相关[可以通过提高可用寄存器组数等方法提高效率(但如果进程数大于组数还是费时)]

3.3 程序运行

3.3.1 进程创建

进程的执行过程可以创建多个新进程 -》 生成一个进程树 process tree

绝大多数系统采用唯一的进程标识符 process identifier pid

父进程可以向子进程传递初始化数据, 当进程创建新进程时, 有2种可能:

  • 父进程与子进程并发执行
  • 父进程等待, 直到某个或者全部子进程执行完

新进程地址空间也有2个可能

  • 子进程时父进程的复制品(完全一样的程序和数据)
  • 加载另外的程序

例: 使用linux下fork()调用后, 复制了原来的地址空间, 父子都将继续执行fork()之后的指令, 不同点在于对于新子进程, 调用fork返回值为0, 父进程为子进程的pid

3.3.2 进程终止

调用exit()进程终止(或者用return隐式调用, 此时进程将返回状态值到父进程(使用wait), 所有子进程的资源将会由系统释放

  • 级联终止 cascade termination: 如果一个进程终止, 其所有子进程也终止

3.4 InterProcess Communication IPC 进程间通信

为什么要支持进程协作

  • 信息共享 information sharing:
  • 计算加速 computation speedup
  • 模块化 modularity
  • 方便 convenience

主要包括2种基本模型 共享内存 shared memory和消息传递 message passing

3.4.1 共享内存系统

缓冲区 buffer (unbounded/bounded)]

in指向下一个空位, out指向第一个满位

  • in==out 空
  • (in+1)%BUFFER_SIZE==out 满

3.4.2 消息传递系统

P&Q进行交流, 他们之间需要有通信链路 communication link

  • 直接或间接通信

    直接-对称性: 指定接收者/发送者: 每个链路只与2个进程有关

    直接-不对称性: 只需指定接收者

  • 同步或异步通信

  • 自动或显式缓冲
